**为什么这片最值得打**：①数量大（一国一篇，能批量产出几十篇）；②买家**带着"怕踩雷"的强意图**主动在查（转化意图高）；③这类"硬知识 + 决策框架"正是 AI 最爱引用、最容易建立**权威复利**的内容；④现在占位的都是泛泛软文，地基松。

## 四、今晚写的内容 + 为什么这么写（每篇都钉在证据上）

> 你特别想看的就是这个：不光给方向，还把内容写出来，并说清"为什么写它 = 对应哪个机会"。

### 📄 `content/drafts/09_europe_client_gift_guide.md`（欧洲篇）
- **为什么写**：今晚新证据显示瑞士(78)/英国(77)/西班牙(77)/北欧(77)/意大利(75)/荷兰**全是品牌空位**，引用全是低权威软文 → 一篇按国别拆、结构化的真权威指南能直接切进去。
- **核心差异化**：抓住欧洲"**反炫耀 + 重合规**"的底色（很多欧洲企业有礼品金额上限）——这是软文不会讲、但买家真正需要的决策框架。
- **诚实做法**：所有国别习俗标"发布前核实"；我方品牌信息留 `[待补]` 占位，**没编**。

### 📄 `content/drafts/10_middleeast_jewish_gift_guide.md`（中东/犹太篇）
- **为什么写**：沙特(77)、以色列都是空位，且带**强禁忌意图**（买家怕踩雷在主动查）。
- **核心差异化**：市面内容常把"中东=穆斯林"一锅烩，**漏掉以色列是犹太教、饮食禁忌逻辑完全不同**（穆斯林忌酒猪/halal；犹太忌贝类、肉奶同食/kosher）。把这层讲清楚 = 别人没占的差异化空位。
- **诚实做法**：宗教/饮食禁忌全标"高风险、发布前逐条核实"；只给红线框架不当确定结论。

## 五、实战暴露的工程问题 + 修复（顺手做了，已测试）

1. **豆包遇到"请求太频繁"(429) 会直接崩**：原程序只对服务器错误(5xx)重试，把 429 当成不可重试 → 第一次跑到第 19 条就崩了。
   **修复**：429 改为"退避后重试"（它本就是"慢点再来"的信号）。`geo/adapters/doubao.py`，并加了回归测试锁住。
2. **整批因为一条失败就全崩**：一条 query 出错不该拖垮整批 100 条。
   **修复**：改成单条容错——撞配额墙优雅停止并报告"已存 N 条、余下明日续跑"，其它错误跳过继续。`geo/recon/batch.py`。
3. **测试**：全套 **35 个测试绿**（在现有 `_is_retryable` 测试里加了 429→可重试 的断言锁住修复）。
