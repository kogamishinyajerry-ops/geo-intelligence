## 5. 策略备忘（Segment A）

**战场**：中文 AI（豆包）+ 货架（抖音商城 / 淘宝）。买家分叉先押 A，英文侧凭证到位再并行验证 B。

**Barbell（两手）**
- **蹭（现金流 + 摸空位，现在能做）**：① 在被豆包高频取货的站点发结构化买家指南——**搜狐系（覆盖最广）+ 新浪 + 知乎/公众号**；② 给信尚礼品这类被引内容供稿/合作，蹭其引用位。
- **自建（自有权威）**：建"高端商务伴手礼"权威内容资产（buying guide + 品牌词 + 跨平台实体一致），目标成为**豆包点名的第一个礼盒品牌**（品牌层空位）；配抖音商城店（4.8+ 评分）+ 淘宝货架。

**里程碑 & 止损线**
- **M1 内容（30 天）**：对 top-4 query 各产 1 篇高权威买家指南，发搜狐/知乎/公众号 → 重跑 recon，目标自有内容进 ≥1 条 query 引用。**止损：60 天 0 进展 → 换 query 或换打法。**
- **M2 品牌（60 天）**：抖音商城店 + 淘宝店上线、评分 4.8+ → 目标豆包某 query 点名我方品牌。
- **M3 监测（每周）**：重跑 12 query，追信尚礼品 vs 我方占答率/引用新鲜度；新对手进答案即标记。

**HITL 闸门**：花钱（供稿/投放/开店）前 + 对外发布前必须人审。
**商品卡**：方舟 API 测不到 → C 端豆包 App 人工抽检 + 抖音店动销数据补。

## 6. 机制发现

- **2/12 query 豆包不联网**（外贸、日本）：从记忆答 → 内容 GEO 杠杆低，宜改走货架直答/品牌词，不浪费内容资源。
- **商品卡：方舟 API 确认不返回**（C 端特性）→ 国内"商品卡→抖音商城"信号需另走 C 端监测线。
- **联网非确定性**：裸 query 高频触发（已加 require_search 重试）；勿加"请联网搜索"系统提示（实测反而抑制）。

## 7. 局限 & 下一步

- **英文侧（Segment B）仍 mock**：给 Perplexity/OpenAI key 即可转真，验证"英文那条更空/客单更高/毛利更厚"的假设。
- **单次时点快照**：联网引用每次略浮动；需 M3 周期性重跑建时间序列（占答率趋势）。
- **适配/供应链/毛利未评**：选品分只含 GEO 可测维度；go/no-go 终裁需补我方能力 + 成本（HITL）。
- **品牌发现仍靠 watchlist**：新对手品牌需扩名单后对存档证据重算（`python -m geo.recon.rederive`）。
